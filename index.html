Our speakers & their talks

  • Felix Geisendörfer
    @felixge debuggable.com Watch his code

    Building an alarm clock with node.js

    No matter what I tried, for years it seemed that there was nothing to get me out of bed in the morning. After having almost given up, one day I realized that node.js was the answer to all my troubles.

    This talk will show you how I used JavaScript to change my life by hooking up a few components with a small node.js application that reliably kicks ass when it comes to getting me out of bed in the morning."

  • Paul Campbell
    @paulca pabcas.com Watch his code

    Why I built whenever and why it's cool

    whenever.js is a teeny javascript library for wrapping your client side behavior in a loving English-sentence based embrace.

    I will talk about why I built whenever, why it's cool, show some examples and show where I've used it in practice.

  • Tiffany Conroy
    @theophani tif.ca Watch her code

    Building a minimal form validation library using the HTML5 form validation API

    Finally, we can add validation rules to our form elements, which provides in-browser validation on submit. But what if we want to use Ajax?

    HTML5 defines DOM interfaces with the components we need to validate the form ourselves. It's easy and fun!

  • Ian Jørgensen
    @ianjorgensen ge.tt Watch his code

    Back to data with pubsub.io

    I'll describe through examples pubsub.io and the applications for the query based live data, i'll go into details of data security in an open publish subscribe system, and scaling techniques like hubs clusters and dynamic query indexes.

    Pubsub.io is an attempt to change the way we publish and consume live data. Our take on pubsub is that the usual channel paradigm is too tightly coupled; instead of forcing things into string-based channels, pubsub.io lets you describe what you're looking for.

  • Irakli Gozalishivil
    @gozala jeditoolkit.com Watch his code

    Asynchronously recursive pure function <3 with lazy streams!

    Streamer library, which suggests opinionated stream definition / pattern and utilizes core library, that makes asynchronously recursive, pure function <3 possible without spaghetti code ninjas!

  • Michiel de Jong
    @unhosted unhosted.org Watch his code

    Announcing the unhosted app store

    The Unhosted project aims to free the web from monopolies. As such, we will be launching an app store, you can submit your unhosted web apps, which you code entirely in client-side javascript, without the need for you to run a server.

    The store will be entirely non-profit, but try to channel money back to the app developer wherever this doesn't clash with our ideals about free software and user empowerment.

  • Michael Aufreiter
    @_mql substance.io Watch his code

    Data.js – Data Manipulation and Graph Persistence for Node.js and the Browser

    Data.js tries to provide an API on top of graph data, that makes it easy to access and group data into objects suitable for both server and client side application tasks.

    By allowing to use the same API to construct your domain objects, both on the client and on the server side, Data.js allows for code (and domain knowledge) reuse. Data.js features a query interface, data-transformations (like clustering), validations, realtime updates as well as a CouchDB backend for persistence.

  • Oleg Podsechin
    @olegpodsechin dailyjs.com Watch his code

    Common Node – A CommonJS compatibility layer for Node.js

    Node.js has captured the attention of early adopters by clearly differentiating itself as being asynchronous from the ground up. Now that server side JavaScript is at the cutting edge of the asynchronous, real time web, it is in a much better position to establish itself as the go to language for synchronous CRUD webapps and gain a stronger foothold on the server.

    The talk will cover the current state of server side JavaScript beyond Node by mapping out the key standards & technologies and their respective positions in the stack. It will introduce Common Node, a synchronous CommonJS compatibility layer using node-fibers which bridges the gap between the different platforms. We'll look intoCommon Node's internals, compare its performance to that of other implementations such as RingoJS and go through some ideal use cases.

    A case will be made for a cross platform, synchronous, Rails like framework. In conclusion, we will try to identify new areas where server side JavaScript could really shine, including API mashups, collaborative development and crowd sourced functionality.

  • Morgan Roderick
    @mrgnrdrck roderick.dk Watch his code

    Closures, this, call and apply – an attempt at an explanation

    Closures, 'this', call and apply are hard topics to grasp for newcomers to JavaScript.

    In this talk I will try to explain (with illustrations) what closures are, what 'this' is and how it relates to closures, how callbacks can be confusing and how confusion can be avoided.

    Once you're comfortable and think you've understood 'this', I'll throw call and apply at you. When we emerge on the other side, there will be unicorns and pixie dust :)

  • Rene Dudfield
    @renedudfield rene.f0o.com Watch his code

    Server side jquery

    Tales of adventures using server side jquery and json instead of a templating language. How server side jquery works, and why server side jquery is useful. Benchmark results comparing this technique to other templating systems. Finally a discussion on how to integrate it with other systems.
